What is the Personal AI Playbook?

The full explanation — reopen this anytime from the ❓ button.

The problem this solves: rewriting the same kind of prompt from scratch every time you need it — a debugging prompt, a LinkedIn caption prompt, a doc-explainer prompt. This tool turns each of those into a reusable workflow: a template with blanks (called variables) that you fill in instead of retyping the whole instruction.

The four sections:

  • Dashboard — your overview: stats, favorites, and shortcuts to jump anywhere.
  • Workflows — your library of saved templates, organized into categories like Code, Learning, Building AI Tools, and Content. Open one, fill in the blanks, copy the result.
  • Prompt Builder — for a one-off prompt you haven't saved yet. You pick building blocks (Role, Objective, Context, etc.), each with a plain-language explanation of what it does, and watch the full prompt assemble live. You can save the result as a new Workflow.
  • Loop Builder — for tasks where one AI response usually isn't good enough on the first try. It wraps any prompt with self-checking instructions (a goal, a checklist, a stop condition) so the AI evaluates and improves its own output across several rounds automatically.

Where your data lives: everything you create is saved in this browser's local storage — nothing is sent anywhere. Use Export to back it up as a JSON file, and Import to restore it on another device.

Keyboard shortcuts: 1–4 switch sections, / jumps to search, n creates a new workflow, ? reopens this panel, Esc closes any open dialog.

When you need something the library doesn't cover, use Prompt Builder to assemble one from scratch, or Loop Builder if you want the AI to self-check and iterate instead of a single one-shot answer.

New workflow

A workflow is a reusable prompt template — write it once with {{variable}} placeholders, fill them in differently every time you use it.

Write your prompt normally, and wrap any part you'll want to swap out each time in double curly braces, like {{language}}. Each unique {{name}} becomes a fill-in field automatically.
