# Day 27 - Claude Challenge

## 📅 Date

June 27, 2026

## 🚀 Project

**Prior Authorization Story Simulator**

## 🎯 Objective

Build an interactive educational web application that teaches the U.S. healthcare Prior Authorization (PA) process through storytelling and guided conversations. The project focuses on making complex healthcare workflows easier to understand using interactive decision-making and conversational learning.

---

## 🛠️ Technologies Used

* HTML5
* CSS3
* Vanilla JavaScript

---

## ✨ Features Implemented

* Interactive story-driven learning experience
* Conversational chat interface
* Guided decision-making system
* Multiple real-world Prior Authorization scenarios
* Branching conversations based on user choices
* Educational explanations throughout the workflow
* Patient, Provider, and Payer perspectives
* Step-by-step healthcare authorization process
* Dynamic story progression
* Modern responsive user interface
* Restart story functionality
* Smooth animations and transitions
* Mobile-friendly design

---

## 📚 What I Learned

* How Prior Authorization works in the U.S. healthcare system.
* Designing conversational user interfaces for education.
* Building branching story logic using JavaScript.
* Creating interactive learning experiences instead of static documentation.
* Managing application state in a single-page application.
* Improving user engagement through storytelling and guided interactions.

---

## 💡 Key Takeaways

This project demonstrated how storytelling can transform complex healthcare workflows into engaging educational experiences. By combining interactive conversations with branching decisions, I learned how to build applications that are both informative and enjoyable to use while strengthening my front-end development skills.
