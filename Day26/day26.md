# Day 26 - Claude Challenge

## 📅 Date

June 27, 2026

## 🚀 Project

**Prior Authorization Workflow Simulator**

## 🎯 Objective

Build an interactive web application that simulates the U.S. healthcare Prior Authorization (PA) workflow through a gamified drag-and-drop experience. The goal was to understand healthcare operations while creating an engaging educational tool.

---

## 🛠️ Technologies Used

* HTML5
* CSS3
* Vanilla JavaScript

---

## ✨ Features Implemented

* Interactive drag-and-drop workflow
* Three workflow lanes:

  * Patient
  * Provider
  * Payer
* Multiple patient scenarios:

  * Elective Surgery
  * MRI
  * Specialty Medication
  * Inpatient Admission
* Medical necessity evaluation
* Prior Authorization document collection
* Submission to payer
* Review outcomes:

  * Approval
  * Pend
  * Denial
  * Appeal
  * Peer-to-Peer Review
* Educational explanations after every step
* Progress tracker
* Days elapsed counter
* Efficiency score calculation
* Celebration animation on approval
* Workflow summary screen
* Restart / New Patient functionality
* Responsive modern UI

---

## 📚 What I Learned

* How the U.S. Prior Authorization workflow operates.
* Designing interactive workflow simulations.
* Implementing drag-and-drop functionality using Vanilla JavaScript.
* Managing application state without external libraries.
* Creating educational user experiences with gamification.
* Building responsive interfaces using only HTML, CSS, and JavaScript.

---

## 💡 Key Takeaways

This project combined healthcare operations with front-end development, demonstrating how complex business workflows can be transformed into engaging interactive learning experiences. It strengthened my understanding of state management, user interaction, workflow visualization, and responsive UI design.
